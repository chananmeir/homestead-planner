# Structure Icons Needed for PropertyDesigner

## Icon Specifications
- **Size**: 512x512 pixels
- **Format**: PNG with transparency

- **Style**: Match existing plant-icons aesthetic

## Summary

- **Total Icons Needed**: 70 PNG files
- **Priority 1**: 15 icons (essential, create first)
- **Priority 2**: 12 icons (ground coverings)
- **Priority 3**: 43 icons (remaining structures)

## Fallback Behavior

The system will automatically use emoji fallbacks for any missing PNG files, so you can:
1. Create Priority 1 icons first and deploy
2. Add Priority 2 and 3 icons incrementally
3. Users will see emojis for structures without PNGs (no broken images)

## Icon Creation Tips

1. **Consistent Style**: Match the existing plant-icons in `frontend/public/plant-icons/`
2. **Clear Silhouettes**: Icons should be recognizable at small sizes
3. **Transparency**: Use alpha channel for rounded edges/shadows
4. **Compression**: Use tools like TinyPNG to keep file sizes under 50KB
5. **Color Palette**: Reference the `color` field in structures_database.py for base colors

## Verification

After creating icons, verify they work:
1. Place PNG files in `frontend/public/structure-icons/`
2. Start the app and drag structures onto the property map
3. Check browser console for 404 errors (missing PNGs)
4. Test fallback by temporarily renaming a PNG file
